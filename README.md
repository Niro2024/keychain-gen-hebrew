# מחולל מחזיקי מפתחות (STL רב-חלקי)

מקלידים שם → מקבלים מודל תלת-ממדי מפוצל לחלקים נפרדים, קובץ STL לכל חלק,
כדי לשייך כל אחד לפילמנט אחר בסלייסר. הכול רץ בדפדפן, בלי שרת.

```bash
npm install
npm run dev      # פיתוח
npm run build    # typecheck + build לתיקיית dist/ (אתר סטטי)
npm run smoke    # בדיקת רגרסיה על רשימת שמות קבועה
```

בדיקת שם בודד ללא UI: `node scripts/smoke.ts "André" Pacifico.ttf`

## מה נבנה

| חלק | גיאומטריה |
|---|---|
| בסיס | offset של מתאר השם החוצה, מאוחד עם לולאת המחזיק לגוף אחד, חור הלולאה חתוך |
| אותיות | extrude ישיר של מתאר האותיות, יושב על פני הבסיס |
| צורה (אופציונלי) | אייקון צמוד לשם, חלק נפרד כדי שאפשר להדפיסו בצבע משלו |
| מסגרת (אופציונלי) | טבעת דקה סביב כל אות, באותו טווח גובה כמו האותיות |

הצינור: `opentype.js` לחילוץ מתארים → `manifold-3d` (Clipper2) לכל ה-CSG →
ייצוא. כל חלק נבדק manifold/watertight לפני שהוא מגיע למשתמש.

## ייצוא

**3MF ([`src/threemf.ts`](src/threemf.ts)) — זו דרך ההורדה המומלצת.** קובץ אחד
שבו כל חלק הוא `<object>` נפרד עם `displaycolor` משלו, וכולם מקובצים תחת
אובייקט-אב אחד דרך `<components>`. סלייסרים (Bambu Studio, OrcaSlicer,
PrusaSlicer) טוענים את זה כאובייקט אחד עם כמה חלקים, מיושרים ומוכנים לשיוך
פילמנטים — במקום כמה STL שצריך לטעון ולסדר ידנית. גם קטן פי 10 בערך.

**STL ([`src/stl.ts`](src/stl.ts))** — ZIP אחד שבתוכו STL נפרד לכל חלק,
לסלייסרים שלא קוראים 3MF. יש גם כפתור לכל חלק בנפרד. אותו קוד ייצוא רץ
בדפדפן ובבדיקות, כך שה-smoke בודק את מה שהמשתמש באמת מקבל.

שני הפורמטים יורדים כקובץ **אחד**. הורדות מרובות רצופות נחסמות ע"י הדפדפן,
וגם מסורבלות בסלייסר.

### ארבע נקודות שאינן מובנות מאליהן

- **גודל הגופן קבוע, האורך נגזר ממנו** — ולא הפוך. מתיחת כל שם לאורך יעד
  אחיד נותנת לשם קצר אותיות ענקיות ולשם ארוך אותיות זעירות: נמדד em של
  112 מ"מ ל-"Ji" מול 18 מ"מ ל-"Maximilian", פי 6. סדרת מחזיקים יוצאת אקראית.
  ה-smoke נועל את זה: `na` ו-`nananana` חייבים לצאת באותו גובה אות בדיוק.
  הרשת בתצוגה (ריבוע = 10 מ"מ) נותנת קנה מידה, כי האורך כבר לא ידוע מראש.

- **גישור אוטומטי** (`bridge` ב-`geometry.ts`) — closing מורפולוגי שרדיוסו גדל עד
  שהבסיס הוא גוף אחד. בלי זה כל אות בעברית, וכל נקודה מעל `i`, יוצאת אי נפרד
  ב-STL והמחזיק מתפרק. רדיוס הגישור בפועל מוצג בשורת הסטטוס.
- **סתימת חורים** — הבסיס הוא לוחית גב; החור היחיד בו הוא של הלולאה. הסתימה
  קורית *אחרי* איחוד הלולאה, כדי לתפוס גם את הסהר שנוצר כשהדיסקה נוגעת
  בבסיס בשתי נקודות (`assert baseGenus === 1` ב-smoke).
- **איחוד הלולאה נעשה בדו-ממד** ולא ב-3D. שני הגופים קופלנריים ובאותו גובה,
  אז התוצאה זהה ל-union תלת-ממדי אבל אין בוליאן תלת-ממדי שיכול להיכשל.

## צורות

83 צורות ([`src/shapes.ts`](src/shapes.ts)) בשש קבוצות — אהובים, חיות, טבע,
אוכל, משחק ותנועה, ומדע וטכנולוגיה. הן גליפים מ-Font Awesome Free Solid, כלומר עוברות בדיוק
באותו pipeline כמו האותיות ולא נדרש בשבילן קוד גיאומטריה חדש. גופן האייקונים
(420KB) נטען רק כשבוחרים צורה.

הצורה היא **חלק נפרד** ב-STL/3MF כדי שאפשר יהיה להדפיס לב אדום ליד שם כתום,
אבל היא נכנסת לצללית המשותפת שממנה נגזרים הבסיס, המסגרת ובדיקת עובי הקו —
אחרת הבסיס לא היה עוטף אותה.

שתי החלטות שנבעו מהסתכלות ולא מבדיקה אוטומטית:

- **13 אייקונים הוסרו אחרי שעברו את כל המבדקים** (dragon, ice-cream, candy-cane,
  bicycle, train, football, spa, satellite, satellite-dish, code, wave-square,
  network-wired, syringe). ברוב מקרי המדע והטכנולוגיה החומר עצמו עבה בהחלט —
  מדידה בסף 0.5 מ"מ נתנה 0% לכל 44 המועמדות — הם פשוט לא נקראים ככלום בגודל
  מחזיק מפתחות. הבדיקה מודדת עובי קו, לא קריאוּת.
- **הפרח פרוצדורלי** (`flowerPolygons`) ולא גליף: ל-FA Free אין פרח, והחלופה
  הייתה גופן אייקונים שני במשקל 348KB בשביל גליף אחד. שישה עיגולים חופפים
  ומרכז עושים את העבודה בעשר שורות.

המרווח בין הצורה לשם נגזר מ-`baseOffset` ולא מגודל הצורה. גזירה מגודל הצורה
השאירה צוואר ארוך שנראה כמו שני אובייקטים קשורים, ודחפה את רדיוס הגישור ל-5.8
מ"מ — מה שניפח גם את הבסיס סביב האותיות.

## גופנים

14 גופנים ב-`public/fonts/`, כולם חופשיים לשימוש מסחרי — 13 תחת OFL-1.1
ו-Satisfy תחת Apache-2.0. ייחוס ונוסח הרישיון: [`public/fonts/LICENSES.md`](public/fonts/LICENSES.md).
מקובצים בתפריט לפי תמיכה
בעברית: 8 עבריים (Secular One, Suez One, Rubik, Varela Round, Fredoka,
Playpen, גברת לוין, Amatic SC) ו-6 לטיניים. טקסט עברי מסודר RTL אוטומטית לפי
אשכולות גרפמים; אם בוחרים גופן לטיני לשם עברי מוצגת התרעה.

**להוספת גופן:** מורידים TTF ל-`public/fonts/`, מוסיפים שורה ל-`src/fonts.ts`,
ומריצים `npm run smoke`. הבדיקה מאמתת כיסוי גליפים, watertight, ומקשה אחת —
אבל *לא* מחליפה הסתכלות: Rubik Bubbles עבר את כל הבדיקות ונפסל רק בעין, כי
בליטות התבנית יוצרות קצוות שלא נקראים בהדפסה.

## מה לא נכנס

מצב batch + ZIP, שמירת פריסטים, העלאת גופנים ע"י המשתמש, וה-slash commands של
`.claude/commands/` — `npm run smoke` כבר מכסה גם `/test-name` וגם
`/batch-smoke`, ובדיקות ה-watertight וה-3MF רצות בתוכו.

## מגבלות ידועות

- אין ליגטורות/חלופות הקשריות: הגליפים נפרסים ידנית (`textPolygons`) כי
  ה-shaper של opentype.js קורס על טבלאות GSUB מסוימות. לשמות זה לא קריטי.
- טקסט מעורב עברית+אנגלית באותה שורה יסודר לא נכון (היפוך מחרוזת מלא, לא bidi אמיתי).
- החישוב רץ ב-thread הראשי. ~25-60 מ״ש לשם, מספיק מהיר; אם זה יגדל — web worker.
